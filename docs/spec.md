# 逆ポーランド記法 (RPN) 電卓の仕様

## 根本的なルール

この電卓では、逆ポーランド記法で記述された数式 (文字列) を受け取りそれを左から右へ計算するものである。

数式は以下のトークンの列として構成される。

- **オペランド (Operand)**: 数値（整数、浮動小数点数）
- **演算子 (Operator)**: 加減乗除などの計算記号（例: `+`, `-`, `*`, `/`）
- **区切り文字 (Delimiter)**: トークン同士を区別するための空白文字

## 使用できる演算子

この電卓では以下に示すような演算子を用いて計算を行うことができる

- `+`: `A B +` と与えられたとき、`A` と `B` の和を計算する
- `-`: `A B -` と与えられたとき、`A` と `B` の差を計算する
- `*`: `A B *` と与えられたとき、`A` と `B` の積を計算する
- `/`: `A B /` と与えられたとき、`A` と `B` の商を計算する

## 詳細な仕様

### 数字が足りない場合

```
1 +
```

のように、演算子の前に 1 つしかオペランドがない場合、「オペランドが足りない」という旨のエラーを出す。

### 演算子が足りない場合

```
1 2 3 +
```

のように、計算した後数字が 2 つ以上残っている場合、「演算子が足りない」という旨のエラーを出す。

### ゼロ除算

```
1 0 /
```

のような入力に対し、「ゼロ除算エラー」として処理を中断する。

### 整数と少数の扱い

```
5 2 /
```

のように、結果が整数とならない商演算について、結果は小数となる (この場合、計算結果は 2.5 である)。

### 無効な文字の扱い

```
1 2 @ +
```

のように、定義されていない文字 (`@` など) が入力された場合、「不明なトークンエラー」として処理を中断する。
