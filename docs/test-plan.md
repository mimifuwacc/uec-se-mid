# 逆ポーランド記法 (RPN) 電卓のテストシナリオ

## 正常系のテストシナリオ

基本演算および仕様で定義された数値（整数・小数）の扱いが正しく機能することを確認する。

- 加算・減算・乗算が正しく行える
- 除算の計算結果について、割り切れるものは整数、割り切れないものは小数になる
- 3 項以上の複合演算が行える
- 小数・負数が含まれていても正しく計算が行える

## 異常系のテストシナリオ

仕様で定義されたエラー条件に対し、適切なメッセージが出力され処理が中断されることを確認する。

- 演算子の前に 1 つしかオペランドがない場合: 「オペランドが不足しています」と出力
- 計算した後数字が 2 つ以上残っている場合: 「演算子が不足しています」と出力
- `1 0 /` のような入力があった場合: 「ゼロ除算エラー」と出力
- `1 2 @ +` のように定義されていない文字が入力された場合: 「不明なトークンエラー」と出力
